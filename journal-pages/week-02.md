## Interactivity 
### Activity 1 - Drawing With Code
In this activity, I learned the basics of using p5.js to create shapes like rectangles, circles, and triangles. It was interesting to see how changing numbers can move shapes or change their size. I also noticed that the order of the code affects which shapes appear on top. This helped me understand how composition works in coding, similar to layering in design. At first it was a bit confusing, but experimenting made it easier to understand.

## Activity 2 - Make an Interactive Sketch

This activity helped me understand how to make my sketch interactive using tools like sliders and buttons. I used a slider to change the size of a shape and a button to randomise its position. It was interesting to see how user input can directly affect what happens on the screen. Sometimes the code didn’t work at first, but fixing it helped me learn how each function works. It made the sketch feel more dynamic and engaging compared to static drawings.

### Activity 3 - Vibe Code an Interactive Sketch
In this activity, I used an LLM to help create a more complex interactive sketch. It was helpful because I could describe what I wanted in simple language and get working code. Some parts worked immediately, while others needed small changes. By reading and editing the code, I started to understand how it works. I learned that LLMs are useful for generating ideas and solving problems, but I still need to understand and adjust the code myself. This activity made coding feel more creative and less difficult.

I translated my hand-drawn data portrait about phone usage into code by identifying key types of data. The time of day and duration are numeric values, while reasons for checking my phone (such as boredom, habit, or important tasks) are categories. My feelings are more qualitative, so I represented them using colours. Instead of showing every detail, I focused on the most important parts: when I check my phone, why I check it, and how I feel. These were the most interesting to make interactive because they show patterns in behaviour.

I created an interactive p5.js sketch where each phone check is shown as a circle on the screen. The position represents time, the size shows how long I used my phone, and the colour represents my feeling. I added a slider to filter the time of day and a dropdown menu to select different reasons (like boredom or habit). This allows the viewer to explore specific parts of my behaviour. Compared to the hand-drawn version, interaction makes it easier to see patterns, such as when I check my phone the most or how my feelings change. When the viewer changes the controls, the visualisation updates immediately, which makes the experience more engaging and dynamic.

When testing my sketch, I noticed that some parts were confusing at first, especially understanding what each colour and size meant. After showing it to others, I improved the clarity by adding labels and making the controls easier to use. I also adjusted the colours to make differences more visible. Through this process, I learned that interaction design is not only about making something work, but also making it easy to understand. This iteration helped me improve both the visual and functional aspects of my data visualisation.


### Documentation and Reflection
I chose to work with data about when and why I check my phone, focusing on time of day, reasons, feelings, and duration. These aspects felt the most meaningful because they reveal patterns in both my behaviour and emotions. In my hand-drawn data portrait, I used visual elements like colour, size, and position to represent this information, so I continued using similar elements in my p5.js sketch to keep a clear connection between the two. Time is shown through horizontal position, feelings are represented by colour and vertical placement, and duration is shown through the size of each shape.

When deciding on interactive elements, I chose a slider and a dropdown menu because they directly relate to the structure of my data. The slider controls time, which is a numeric value, allowing the viewer to filter different parts of the day. The dropdown menu filters the reasons for checking my phone, which are categories such as boredom, habit, or important tasks. I wanted the interaction to be simple and intuitive, so users can easily explore the data without confusion.

By interacting with the sketch, viewers can learn more than they could from the hand-drawn portrait. In the drawing, all the data is presented at once, which can feel crowded and harder to analyse. In contrast, the interactive version allows users to isolate specific patterns, such as only looking at times when I felt stressed or when I checked my phone out of habit. This makes it easier to compare and understand behaviours over time. The immediate response of the sketch when controls are changed also makes the experience more engaging.

I used vibe coding with the help of an LLM to build and improve my sketch. This allowed me to create features that I did not fully know how to code myself, such as filtering data and adding interactivity. However, I still needed to read, test, and adjust the code to make sure it worked correctly. Through this process, I learned how different parts of the code connect and how small changes can affect the outcome.

If I had more time, I would develop the visual design further to make it more expressive and closer to my hand-drawn style. I would also add smoother transitions and more detailed interactions. Overall, this project showed me that data can be both analytical and creative, and that combining hand-drawn and digital approaches can create a more meaningful and engaging experience.
